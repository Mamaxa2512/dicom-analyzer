# Керівництво з реалізації: DICOM Medical Image Analyzer

Цей проект — веб-додаток для аналізу медичних зображень у форматі DICOM. Він розроблений спеціально для того, щоб підсилити твоє резюме при подачі на позицію Software Development Student у Siemens Healthineers, продемонструвавши розуміння їхньої предметної області (медичні зображення, сканери, формати даних).

## 📂 Структура проекту (Скелет вже створено)

```
dicom-analyzer/
├── app/                        # Основний код бекенду (Python/Flask)
│   ├── __init__.py             # Ініціалізація Flask-додатку
│   ├── routes.py               # Маршрути (REST API) для обробки запитів
│   ├── dicom_parser.py         # Логіка читання DICOM файлів (pydicom)
│   └── image_processor.py      # Фільтри та обробка зображень (numpy, scipy)
├── static/                     # Статичні файли для фронтенду
│   ├── css/
│   │   └── style.css           # Стилі (рекомендується темна "медична" тема)
│   ├── js/
│   │   └── viewer.js           # Логіка відображення та інтерактивності (Canvas)

├── templates/                  # HTML-шаблони (Фронтенд)
│   ├── base.html               # Базовий шаблон (навігація, структура)

│   └── viewer.html             # Сторінка перегляду та аналізу зображень
├── tests/                      # Папка для тестів (pytest)
│   └── sample_data/            # Зразки файлів для тестів
├── sample_dicoms/              # Демонстраційні DICOM-файли (для показу проекту)
├── PROJECT_GUIDE.md            # Цей файл
├── requirements.txt            # Залежності проекту
```

## 🛠️ Кроки реалізації (Що тобі потрібно запрограмувати)

Проект розбитий на 3 логічні фази. Тобі потрібно поступово заповнювати пусті файли-скелети логікою.

### Фаза 1: Базовий функціонал (MVP - Minimum Viable Product)
**Мета:** Навчити додаток приймати DICOM файл, читати його метадані та відображати зображення.

1.  **Налаштування оточення:**
    *   Відкрий цей проект у своїй IDE (наприклад, VS Code або IntelliJ).
    *   Створи віртуальне оточення: `python -m venv venv`
    *   Активуй його: `source venv/bin/activate` (Linux/Mac)
    *   Встанови залежності: `pip install -r requirements.txt`

2.  **`app/dicom_parser.py` (Читання даних):**
    *   Використай бібліотеку `pydicom` для функції `parse_dicom()`, щоб витягти інформацію про пацієнта (ім'я, вік, стать) та знімок (модальність - CT/MRI, дата).
    *   Напиши функцію для витягування піксельних даних (`pixel_array`) та перетворення їх у зрозумілий для вебу формат (наприклад, PNG за допомогою Pillow або base64).

3.  **`app/routes.py` (Маршрутизація):**
    *   Напиши ендпоінт для `/` (рендерить `index.html`).
    *   Напиши POST ендпоінт `/upload`, який зберігатиме файл у `static/uploads/`, викликатиме `parse_dicom()` і повертатиме JSON з даними або рендеритиме `viewer.html` з цими даними.
    *   Напиши ендпоінт для віддачі самого зображення.

4.  **`templates/index.html` (Завантаження):**
    *   Зроби просту форму для завантаження файлу (тип `file`, accept `.dcm`).
    *   (Опціонально) Зроби зону drag-and-drop.

5.  **`templates/viewer.html` (Перегляд):**
    *   Розділи сторінку на дві частини: збоку — список метаданих пацієнта/знімку, по центру — тег `<img>` або `<canvas>` для відображення знімку.

### Фаза 2: Обробка зображень (Image Processing)
**Мета:** Додати медичні фільтри, щоб показати глибоке розуміння того, як працюють діагностичні інструменти.

1.  **`app/image_processor.py` (Логіка фільтрів):**
    *   Напиши функції, які приймають масив пікселів (numpy array) і застосовують до нього фільтри:
        *   `apply_window_level()`: Реалізуй зміну яскравості та контрасту (Window Center / Window Width) — це найважливіша фіча для радіологів (HU Windowing).
        *   `apply_clahe()`: Локальне покращення контрасту (Contrast Limited Adaptive Histogram Equalization).
        *   `apply_edge_detection()`: Виділення контурів органів (наприклад, фільтр Собеля).
        *   `apply_invert()`: Інверсія кольорів (негатив).

2.  **`app/routes.py` (АПІ для фільтрів):**
    *   Створи ендпоінт `/api/process`, який приймає назву фільтра (і параметри W/L), викликає відповідну функцію з `image_processor.py` і повертає нове зображення.

3.  **`static/js/viewer.js` та `templates/viewer.html` (Інтерфейс інструментів):**
    *   Додай кнопки для різних фільтрів.
    *   Реалізуй зміну Window/Level за допомогою перетягування миші по зображенню (якщо використовуєш `<canvas>`).

### Фаза 3: Аналітика та Полірування
**Мета:** Зробити проект професійним та готовим до показу в портфоліо.

1.  **Аналітика (`image_processor.py`):**
    *   Додай функцію для розрахунку гістограми розподілу інтенсивності пікселів.
    *   Виведи цю гістограму на `viewer.html` (можна використати Chart.js).

2.  **Стилізація (`static/css/style.css`):**
    *   Оформи додаток у "клінічному" стилі: темний фон (темно-синій/сірий), чіткі контрастні шрифти, як у реальних медичних програмах (наприклад, Siemens syngo).

3.  **Тестування та Документація:**
    *   Напиши пару простих тестів у `tests/`, щоб перевірити, чи правильно парсяться метадані з тестового файлу.
    *   Напиши гарний `README.md` (я можу допомогти з цим пізніше), де будуть скріншоти, опис технологій та як запустити проект.

## 🔍 Де взяти тестові файли (DICOM)?

Для розробки тобі знадобляться реальні (але анонімізовані) знімки.
*   **Найпростіший варіант:** В самій бібліотеці `pydicom` є тестові файли. У консолі Python напиши:
    `import pydicom.data; print(pydicom.data.get_testdata_files('*.dcm'))`
*   **Для гарного демо:** Завантаж кілька наборів з [OsiriX DICOM Image Library](https://www.osirix-viewer.com/resources/dicom-image-library/) (там є цілі серії CT та MRI). Збережи їх у папку `sample_dicoms/`.

Удачі з розробкою! Якщо знадобиться допомога з реалізацією конкретної функції — звертайся.
